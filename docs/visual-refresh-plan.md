# Visual Refresh Plan

## 1. 現在のデザイン課題
- 既存デザインは機能を満たしているものの、老舗青果流通企業としての信頼感・専門性・歴史を感じさせる重厚感に欠けている。
- ページ構成やコンポーネントが統一されておらず、各ページでレイアウトやヒーローの高さがバラバラ。

## 2. 添付HTMLから採用する要素
- 配色の考え方（深緑、生成り、薄いグレーをベースに、アクセントとしての金色）
- Bento Grid（とくに`/advantage/`のレイアウトに活用）
- 写真の見せ方（16:9, 4:3, 暗めのオーバーレイ）
- セクション構成や控えめなアニメーション
- ヘッダーの半透明/blur表現

## 3. 採用しない要素
- **誤情報・架空情報の全般**: 1954年設立、架空の拠点、架空の数値、架空の技術、ミシュラン店などの固有名詞、AI物流等の表現。
- 外部の一時的な画像URL
- Tailwind CDNや直接のstyleタグ、IntersectionObserverの重複実装などのフロントエンド技術的負債。

## 4. ページ別の変更予定
- **トップページ**: Heroの刷新（画面高70〜85%）、会社紹介、Bento Gridを用いた事業紹介、拠点、ニュース、採用への導線。
- **`/advantage/`**: 荷受け、仲卸、加工、テストキッチンなどのBento Grid化。
- **`/about/`**: 会社紹介、代表メッセージ（既存文面・画像維持）、概要を洗練されたレイアウトへ。
- **`/locations/`**: 確定拠点のみの写真カード型レイアウト。
- **`/recruit/`**: 既存の採用情報（大田、大阪）の職種を維持したスタイリッシュなカード配置。
- **`/contact/`**: 使いやすさを優先したフォーム。
- **`/news/`**: ゼロ件時の品位ある空状態メッセージ。

## 5. 共通コンポーネント
`src/components/ui/` に以下の共通コンポーネントを整備。
- `SectionHeader.tsx`
- `PageHero.tsx`
- `BentoCard.tsx`
- `Reveal.tsx` (CSSアニメーションベース)

## 6. カラートークン
`src/app/globals.css` を更新。白・生成り(55~65%)、薄いグレー(15~20%)、深緑(10~15%)、淡い緑(5~10%)、金色(3~5%)の割合。

## 7. フォント
- 本文・一般見出し: `Noto Sans JP`
- 英数字: `Inter`
- 代表メッセージや歴史に関する限定的見出し: `Noto Serif JP`

## 8. アニメーション
CSSでのフェードイン＆上方向スライド（`opacity`, `translateY`）。`prefers-reduced-motion` 対応必須。

## 9. 画像方針
`public/images/approved/` などの実画像を使用。必要に応じてプレースホルダーを置くが本番非表示。アスペクト比固定、角丸4〜8px。

## 10. 影響ファイル
- `src/app/globals.css`
- `tailwind.config.ts` (Tailwind v4なら CSS変数中心だが、設定ファイルの確認)
- 全ページの `page.tsx` および `components` 類

## 11. ロールバック方法
git commitによるバージョン管理を活用し、必要に応じてリセットする。

## 12. テスト項目
- `npm run lint`, `npm run build`, `npx tsc --noEmit`
- レスポンシブ確認 (375px, 768px, 1024px, 1440px)
- アクセシビリティ・Lighthouse確認
- 未確認情報・架空情報が含まれていないかの目視確認
